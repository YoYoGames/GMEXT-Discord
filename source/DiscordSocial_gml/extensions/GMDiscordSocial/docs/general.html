DiscordErrorType

Discord Enum: discordpp::ErrorType

This enum represents the various types of errors the SDK returns.

These constants are referenced by the following structs:

  • DiscordClientResult


Member Description
None No error, the operation was successful.
NetworkError The user is offline or there was some network issue that prevented an underlying HTTP call from succeeding.
HTTPError An HTTP call was made to Discord's servers but a non success HTTP status code was returned. In some cases this may be retryable, and if so ClientResult::Retryable will be true. In most cases though the failure is due to a validation or permissions error, and the request is not retryable. ClientResult::Error and ClientResult::ErrorCode will have more information.
ClientNotReady An operation such as sending a friend request or joining a lobby was attempted but the Client is not yet ready. Wait for Client::Status to change to Client::Status::Ready before trying again. Also be sure to call Client::Connect to begin the process of connecting to Discord's servers, otherwise the Client will never become ready.
Disabled An operation was temporarily disabled for stability reasons.
ClientDestroyed The Client has been destroyed and so this operation cannot complete.
ValidationError Used when an SDK method is called but the inputs don't pass local validation. For example if one attempts to accept a friend request when there is no pending friend request for that user, this ErrorType would be used. The specific validation error will be included in the error field, and no other ClientResult fields will be set.
Aborted The user or developer aborted an operation, such as an authorization flow.
AuthorizationFailed An authorization function failed, but not necessarily as the result of an HTTP call that returned an error.
RPCError An RPC call was made to Discord's desktop application, but it returned a non-success result. The error and errorCode fields should both be set with more information.

DiscordActivityInvite

Discord Class: discordpp::ActivityInvite

When one user invites another to join their game on Discord, it will send a message to that user. The SDK will parse those messages for you automatically, and this struct contains all of the relevant invite information which is needed to later accept that invite.

This struct is referenced by the following functions:

  • discord_social_client_accept_activity_invite
  • discord_social_client_send_activity_join_request_reply
  • discord_social_client_set_activity_invite_created_callback
  • discord_social_client_set_activity_invite_updated_callback


Member Type Description
type ActivityActionTypes The type of invite that was sent.
sender_id Real The user ID of the user who sent the invite.
application_id Real The target application of the invite.
parent_application_id Real The application ID of the parent - this is only applicable if there is a parent for a publisher's suite of applications.
channel_id Real The ID of the Discord channel in which the invite was sent.
message_id Real The ID of the Discord message that contains the invite.
is_valid Boolean Whether or not this invite is currently joinable. An invite becomes invalid if it was sent more than 6 hours ago or if the sender is no longer playing the game the invite is for.
party_id String The ID of the party the invite was sent for.
session_id String The session ID of the user who sent the invite.

DiscordMessageHandle

A MessageHandle represents a single message received by the SDK.

This struct is referenced by the following functions:

  • discord_social_client_get_message_handle
  • discord_social_client_get_lobby_messages_with_limit
  • discord_social_client_get_user_messages_with_limit


Member Type Description
id Real The ID of this message.
author DiscordUserHandle The UserHandle for the author of this message.
author_id Real The user ID of the user who sent this message.
channel DiscordChannelHandle The ChannelHandle for the channel this message was sent in.
channel_id Real The channel ID this message was sent in.
disclosure_type DiscordDisclosureTypes If this is an auto-generated message that is explaining some integration behavior to users, this will hold the type of disclosure so you can customize it.

edited_timestamp Real The timestamp in millis since the epoch when the message was most recently edited.
